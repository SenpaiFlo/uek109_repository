Cloud

Was ist die Cloud

Die Cloud ist eigentlich ein ausgelagerter "Ort", welcher bei einem Cloud anbieter im Rechenzentrum liegt. Physisch existiert er zwar, jedoch hat man keinen Zugriff darauf. Die Services sind alle virtuell und nicht direkt auf einem Host.

NIST definiert Cloud Computing folgendermassen: Cloud-Computing ist ein Modell, das es erlaubt, bei Bedarf jederzeit und überall bequem über ein Netz auf einen geteilten Pool von konfigurierbaren Rechnerressourcen

Cloud Computing hat 5 Merkmale, welche besonders hervorstehen:

  1. On-Demnad Self Service: Nutzer können die Bereitstellung von Ressourcen selbst managen
  2. Broad network access: Man kommt von überall auf der Welt und mit allen möglichen Rechnern auf die Ressourcen.
  3. Resource Pools: Die Ressourcen des Cloud Providers stehen in verschiedenen Geozonen und diese sind miteinander verbunden.
  4. Rapid Elasticy: Die Ressourcen werden je nach bedarf angepasst. Wenn eine Auslastung höher wird, werden automatisch mehr Ressourcen hinzugefügt. Wenn weniger Ressourcen gebraucht werden, als assigned, werden die Ressourcen auch verringert, aber in standby behalten. Man kann die verwendeten Ressourcen auch an gewisse Schematika anpassen, z. B. an verschiedene Tageszeiten.
  5. "Pay what gets used": Man bezahlt nur für die Ressourcen, welche effektiv gebraucht werden. Dieser Punkt steht direkt in Verbindung mit Rapid Elasticy!

Cloud Dienstleistungen

Es gibt viele verschiedene Cloud Services. Dazu gehören Domain Server/ Network management, FTP Server, Webserver, Storage services, Blob storage, application Server, IoT Services, AI Services, etc.

Cloud Anbieter:

Zu den bekanntesten Cloud Anbietern gehören Microsoft mit MS Azure, Amazon Web Services, Google Cloud, IBM Cloud und noch viele mehr

Cloud Deployment Modelle

Es gibt verschiedene Cloud Modelle. Diese wären die Public Cloud, Private Cloud, Community Cloud und die Hybrid Cloud

Cloud Service Modelle

Cloud Service modelle sagen aus, wie viel der Infrastruktur in der Cloud und wieviel davon noch lokal ist. Dabei gibt es Layer:

  1. On-Premise: Es ist noch alles Lokal und nichts wird von einem Cloud Service gemanaged.
  2. IaaS (Infrastructure as a Service): Die Virtualisierung, Serverdiesnte, Speicherdienste und das ganze Networking steht nun in der Cloud, alles andere ist nach wie vor lokal.
  3. PaaS(Platform-as-a-Service): Nun sind nur noch die Applikationen Lokal gemanaged, sowie die Application Data. Ansonsten wird alles in die Cloud ausgelagert.
  4. SaaS(Software-as-a-Service): Hier wird alles in der Cloud gemacht. Beim Start eines Clients, wird zuerst zu den Servern in der Cloud verbunden, wo er dann alle Daten, etc. bekommt, damit man dann weiterarbeiten kann.

Warum dienste aus der Cloud beziehen?

Die Vorteile sind einfach. Durch die Cloud spart man sich Hardware kosten, denn diese muss man nicht mehr kaufen. Zudem bezahlt man keinen IT-Service Techniker, welcher sich immer wieder um die Lokalen Server kümmern muss. Denn der Techniker im Rechenzentrum ist im Preis schon inbegriffen. Zudem ist es umweltbewusster und es ist performanter.
